# 잠실 테니스장 추가 & 빈자리 현황 요약 알림 — 설계서

> 작성일: 2026-06-23
> 한 줄 요약: ① 잠실 유수지 테니스장 빈자리 알림 추가(송파와 같은 esongpa 시스템), ② "지금 비어있는 자리"를 매일 1번 요약 알림하는 기능 추가.

---

## 배경 (왜 만드나)

- 현재 봇은 강남(포이·세곡) + 송파 테니스장의 **"새로 생긴 취소표"**만 실시간 알림한다.
- 사용자 요청 2가지:
  1. **잠실 유수지 테니스장**도 똑같이 빈자리 알림에 추가. (ID/비번은 송파와 동일)
  2. "새로 생긴 것"이 아니어도 **그냥 지금 비어있는 자리**도 알림받고 싶다.
     (예: 세곡 3번코트 7/2 21시는 봇이 처음 켜졌을 때부터 비어 있어서 지금까지 한 번도 알림이 안 왔다.)

## 기능 ① 잠실 유수지 테니스장 추가

### 현재 구조
- `src/songpa.py` 가 송파 전용으로 하드코딩됨:
  - 주소 `spc.esongpa.or.kr`, 목록 페이지 `s05.od.list.php`
  - 로그인(`/bbs/login_check.php`) → HTML 받기 → 정규식으로 `예약가능`(class=`status_y`) 추출

### 잠실의 차이 (실측)
| | 송파(기존) | 잠실(추가) |
|---|---|---|
| 주소 | `spc.esongpa.or.kr` | `club.esongpa.or.kr` |
| 목록 페이지 | `s05.od.list.php` | `s07.od.list.php` |
| 로그인 | `/bbs/login_check.php` | **확인 필요(구현 1단계)** |
| 빈자리 HTML | `status_y` + `예약가능` | **확인 필요(구현 1단계)** |
| ID/비번 | `SONGPA_ID`/`SONGPA_PW` | **동일(공용)** |

> 로그인 없이 잠실 페이지를 열면 메뉴만 나오고 예약표가 없음 → **로그인 필수 확정**.
> 같은 esongpa 플랫폼이므로 로그인/HTML 구조가 송파와 같을 가능성이 높지만, **반드시 구현 첫 단계에서 실제 로그인 후 HTML을 확인**한다. (로컬 PC엔 송파 ID/비번이 없으므로, GitHub Actions 임시 디버그 실행 또는 사용자가 `.env`에 임시 입력해 확인)

### 설계 (방법 A — esongpa 공용화)
- `songpa.py` 의 로그인·월별조회·파싱 로직을 **시설에 독립적인 공용 함수**로 정리하고,
  시설별 차이(주소·페이지·시간필터·코트라벨)는 **설정 목록**으로 분리한다.

```
# 예시 구조 (구현 시 확정)
ESONGPA_SITES = [
  {center:"송파", base:"https://spc.esongpa.or.kr",  list_page:"s05.od.list.php", wanted: is_songpa_wanted},
  {center:"잠실", base:"https://club.esongpa.or.kr", list_page:"s07.od.list.php", wanted: is_jamsil_wanted},
]
fetch_esongpa_slots():
  로그인 1회(세션 공유 가능하면 공유, 도메인 다르면 시설별 로그인)
  for 시설 in ESONGPA_SITES:
     월별(이번달·다음달) 목록 HTML → 파싱 → 미래 빈자리만
```

- **기존 송파 동작이 깨지지 않도록** 회귀 테스트로 보호한다(같은 입력 HTML → 같은 결과).
- 만약 잠실 HTML이 송파와 미세하게 다르면, **파싱 부분만 시설별로** 분기(설정에 파서 지정).

## 기능 ② 빈자리 현황 요약 알림 (신규)

### 핵심 개념
- 기존 "취소표(새로 생긴 빈자리) 즉시 알림"은 **그대로 유지**.
- 추가로, **매일 1번** "지금 내가 원하는 시간대에 비어있는 자리 전체"를 **요약 1통**으로 보낸다.
- 5분마다 반복 발송하지 않으므로 **알림 폭탄 없음**.

### 동작
- 시각: **매일 아침 8시(KST)** 1회. (확정)
- 내용: 현재 빈자리 중 **원하는 시간대 필터를 통과한 것 전체**(강남 저녁 + 송파/잠실 해당 시간), 시설·코트·날짜·시간 목록.
- 비어있는 게 하나도 없으면: **"현재 빈자리 없음" 한 줄을 발송**(봇이 살아있다는 안심 신호). (확정)

### 구현 방향 (상세는 공정표에서)
- 별도 진입점 + **별도 일일 워크플로**(`schedule: 23:00 UTC = 08:00 KST`)로 책임 분리.
  - 5분 워크플로 = 취소표 즉시 알림(기존)
  - 일일 워크플로 = 빈자리 요약(신규)
- 요약은 "현재 상태"만 읽어 보내므로 **직전 기록(state) 불필요** → 단순·안전.

## 시간 필터 정리 (`filters.py`)

| 시설 | 알림 시간대 | 비고 |
|---|---|---|
| 강남(포이·세곡) | 매일 저녁 19~21시 시작 | 기존 유지 |
| 송파 | 토요일 08·10시 | 기존 유지 |
| **잠실(신규)** | **매일 저녁 19~21시 + 토요일 08·10시** | 신규 `is_jamsil_wanted` |

> ⚠️ 잠실에 실제로 '저녁' 시간칸이 있는지는 구현 1단계에서 확인. 없으면(낮만 운영) 사용자와 재상의.

## 바뀌는 파일 (예상)

| 파일 | 변경 |
|---|---|
| `src/songpa.py` → `src/esongpa.py`(또는 일반화) | 송파·잠실 공용화 |
| `src/filters.py` | `is_jamsil_wanted` 추가 |
| `src/notifier.py` | 빈자리 요약 메시지 포맷 `format_summary` 추가 |
| `src/main.py` | 일일 요약 진입점 추가 (기존 취소표 흐름은 유지) |
| `.github/workflows/` | 일일 요약 워크플로 추가 |
| `tests/` | 잠실 파싱·잠실 필터·요약 포맷·**송파 회귀** 테스트 |

> 규칙 I(파일·함수 작게, 중복 금지) 준수: 공용화로 중복 제거, `main.py` 함수는 한 가지 일만.

## 안전 / 되돌리기 계획

- **데이터 안전:** 이 봇은 회사 재고 DB와 무관(개인용 별도 repo). DB 변경 없음. 읽기 전용 조회 + 텔레그램 발송뿐.
- **기존 동작 보호:** 송파 공용화는 회귀 테스트로 기존 결과 동일함을 보장한 뒤 적용.
- **되돌리기:** 모든 변경은 git 커밋 단위. 문제 시 직전 커밋으로 되돌리면 됨(`git revert`). 잠실/요약 기능은 끄기 쉽게(설정 목록에서 잠실 한 줄 제거, 요약 워크플로 비활성).

## 범위에서 제외 (YAGNI)
- 빈자리 요약의 시각/횟수 사용자 설정 UI — 우선 코드 상수로 고정(아침 8시).
- 잠실 외 추가 시설 — 이번 범위 아님(단, 공용화로 다음에 쉽게).
- 코트별 세분화 필터 — 페이지가 주는 전체를 시간필터만 적용.

## 구현 순서 (개략 — 상세는 공정표)
1. **잠실 실제 구조 확인** (로그인 후 HTML) → 송파와 동일 여부 판정
2. esongpa 공용화 + 송파 회귀 테스트
3. 잠실 설정·필터 추가 + 테스트
4. 빈자리 요약 포맷·진입점 + 테스트
5. 일일 요약 워크플로 추가
6. 로컬 검증 → 사용자 승인 → 1층(main) 배포
