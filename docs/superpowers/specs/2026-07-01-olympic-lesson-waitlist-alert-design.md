# 올림픽공원 테니스 레슨 대기 알림 — 설계서

> 작성일: 2026-07-01
> 한 줄 요약: 국민체육진흥공단(ksponco) **올림픽공원 테니스 레슨 대기 현황표**에서
> **주중(평일) 19시 실외·실내** 두 칸을 지켜보다가, 대기 줄이 열리거나(마감→숫자)
> 다시 닫히면(숫자→마감) 텔레그램으로 각각 1통씩 알린다.
> 기존 강남·송파·잠실·대치유수지와 **같은 봇·같은 알림창**에서 함께 본다.

---

## 1. 목적

올림픽공원 테니스 레슨은 인기가 많아 대기 줄까지 **마감**되면 대기 신청조차 못 한다.
누군가 빠져 대기 줄이 열리면 현황표의 해당 칸이 **'마감' → 숫자**로 바뀐다.
그 순간을 놓치지 않고 알려서 **바로 대기 신청**할 수 있게 한다.
다시 마감되면(숫자→마감)도 1통 알려, 창이 닫혔음을 알 수 있게 한다.

## 2. 확정 요구사항 (사용자 협의 완료)

| 항목 | 내용 |
|------|------|
| 감시 대상 | **주중(평일) 19시**, **실외·실내 둘 다** (하나라도 열리면 알림) |
| 시간 | **19시만** (저녁 7시). 주변 시간(18·20시)은 지금 안 함 |
| 알림 규칙 | **마감→숫자 = 1통(대기 열림)**, **숫자→마감 = 1통(다시 마감)** |
| 반복 | 상태가 그대로면 조용(열려 있는 동안 5분마다 재알림 안 함) |
| 동작 | **읽기 전용** — 자동 대기신청 안 함 (강남 등과 동일) |
| 알림창 | 기존 텔레그램 봇에 합류(별도 알림창 X) |

> **'주중'은 표의 한 줄**이다 — 월~금을 하나로 묶은 값이라, "평일 19시"는 곧
> `(주중, 실외, 19시)`·`(주중, 실내, 19시)` **두 칸**을 뜻한다.
> `숫자`의 정확한 의미(대기 인원/자리 수)는 사이트만 알지만, **어떤 숫자든 = 대기 신청 가능**이므로
> "숫자면 열림, 마감·X면 닫힘"으로만 판정한다.

## 3. 사이트 분석 결과 (실측 2026-07-01)

- 조회 주소: `https://www.ksponco.or.kr/spm/reservationStatus/tennis/waitList.do?textSize=normal`
- **로그인 불필요** — 누구나 대기 현황표를 볼 수 있음 (강남·대치유수지와 같은 급)
- **서버 렌더링 HTML** — 표가 페이지에 바로 들어있음 (JavaScript/AJAX 아님, 첫 GET에 다 옴)
- **날짜·시설 선택칸 없음** — `<select>`·달력·날짜 파라미터 전무 →
  **고정 주간 시간표**(주중/주말 반복)라 날짜 축이 없음. `waitList.do` 하나가 곧 올림픽공원 표.
- 표 구조(TR 6줄, 각 줄 정확히 **17칸** = `[요일, 코트, 6시…20시(15칸)]`, **rowspan 없음**):

  | 요일 | 시간/코트 | 6시 | 7시 | … | **19시** | 20시 |
  |------|-----------|-----|-----|---|----------|------|
  | 주중 | 실외 | 마감 | 마감 | … | **마감** | 마감 |
  | 주중 | 실내 | 마감 | 마감 | … | **마감** | 마감 |
  | 주말 | 실외 | X | 마감 | … | 마감 | X |
  | 주말 | 실내 | 마감 | … | … | 마감 | 마감 |
  | 수요일 | 실내 | 마감 | … | … | X | X |

  - 첫째 칸(`<th>`) = **요일**(`주중`/`주말`/`수요일`), 둘째 칸(`<th>`) = **코트**(`실외`/`실내`)
  - 셋째 칸부터 시간칸 15개. 머리줄(TR0)의 `6시`…`20시`에서 숫자만 뽑아 **시간→열번호** 지도를 만든다
    (열 위치가 바뀌어도 안전).
  - 칸 값 3종: **`마감`**(대기 마감) / **숫자**(예: `19`, `17` — 대기 가능) / **`X`**(레슨 없음)

## 4. 접근법 — 전용 부품 신설

봇은 "**시설군마다 자기 부품 하나**" 구조다(강남=`fetcher.py`, 송파·잠실=`esongpa.py`,
대치유수지=`daechi.py`). 올림픽공원은 사이트도 다르고, 다루는 대상도 다른 종류다:

- 강남·송파·잠실·대치유수지 = **코트 빈자리(취소표)** → `Slot`(코트+날짜+시간) 목록 흐름
- 올림픽공원 = **레슨 대기 칸의 상태 변화(마감↔숫자)** → 날짜 없는 **칸별 상태 전이**

성격이 달라 기존 `Slot`/빈자리 흐름에 억지로 끼우면 알림 문구가 "빈자리 현황"과 섞여 헷갈린다.
그래서 **`src/olympic.py`** 를 새로 만들어 나란히 두고, **자기 전용 알림 문구**로 보낸다.
발상은 기존 `find_opened`(신청 준비중→접수중 전이 감지)와 같되, **양방향**(열림·닫힘)이다.

## 5. 컴포넌트 설계

### 5.1 `src/olympic.py` (신규)

**(a) `parse_olympic(html, targets) -> dict`** — 감시 칸의 현재 상태만 추출(순수 파싱)
- `targets`: 감시할 `(요일, 코트, 시)` 목록. 예: `[("주중","실외",19), ("주중","실내",19)]`
- 표를 TR로 나눠, 머리줄에서 `{시간: 열번호}` 지도를 만든다(`N시`에서 숫자만 추출).
- 각 데이터 줄에서 `요일`(0번칸)·`코트`(1번칸)를 읽고, 감시 대상이면 해당 시각 열의 칸 값을 본다.
- 반환: `{"주중 실외 19시": "열림:19", "주중 실내 19시": "닫힘"}` 형태의 dict.
  - 칸 텍스트가 **숫자**면 `"열림:<숫자>"`, 아니면(`마감`·`X`) `"닫힘"`.
  - (숫자를 붙여두면 알림 문구에 "지금 19" 같이 보여줄 수 있고, 상태 비교에도 문제없다 —
    열림 상태가 유지되는 한 숫자만 바뀌어도 '열림'이면 재알림하지 않도록 7절에서 정규화.)

**(b) `fetch_olympic_states(settings) -> dict | None`** — 실제 조회 + 파싱
- `settings["올림픽공원레슨"]`에서 `받기`·`코트`·요일별 시각을 읽어 `targets`를 만든다.
- `받기=False`/설정 없음/대상 0개 → **빈 dict `{}`**(감시 끔, 조회 안 함).
- 페이지 GET(로그인 없음, `make_session`+타임아웃) → `parse_olympic` → 상태 dict 반환.
- **조회·파싱 실패 시 `None`** 반환(빈 dict와 구분) → main이 "직전 상태 유지 + 실패 누적"으로 처리.

**(c) `build_targets(cfg) -> list[tuple]`** — 설정 → 감시 칸 목록(순수함수)
- `cfg = {"받기": True, "코트": ["실외","실내"], "주중": [19]}` 형태.
- 요일키(`주중`/`주말`/`수요일`) × `코트` × 시각 → `(요일, 코트, 시)` 튜플들.
- 순수함수라 단위 테스트가 쉽다(설정 조합 → 정확한 대상 목록).

상수(파일 상단): `OLYMPIC_URL`(config에서 import), `HEADERS`, `REQUEST_TIMEOUT = 8`,
`DAY_KEYS = ("주중", "주말", "수요일")`, `COURT_KEYS = ("실외", "실내")`.

### 5.2 `settings.yaml` — 올림픽공원레슨 블록 추가

```yaml
올림픽공원레슨:            # 국민체육진흥공단 올림픽공원 테니스 레슨 대기
  받기: true              # 끄려면 false 로만 바꾸면 끝
  코트: [실외, 실내]        # 감시할 코트(실외/실내). 둘 다면 둘 다 적기
  주중: [19]              # 평일 저녁 7시. 나중에 [18, 19, 20]처럼 늘리기 가능
  # 주말: [10]            # (선택) 주말도 보려면 주석 지우고 시각 적기
```

### 5.3 `src/settings_loader.py` — 올림픽 블록 특례 검증

이 블록은 기존 시설(빈자리)과 **모양이 다르다**(`코트` 키 = 글자 목록, 요일키 = `주중`/`주말`/`수요일`).
기존 `validate_settings`는 시간키를 `요일(월~일)·매일·평일·주말`로만 허용하므로 그대로면 거부된다.
→ `코트추가`처럼 **`올림픽공원레슨` 키를 특례 분기**로 처리한다.

- 새 함수 `_validate_olympic_block(cfg)`:
  - `받기`: bool
  - `코트`: 리스트이며 각 원소가 `실외`/`실내` 중 하나
  - 그 외 키: `주중`/`주말`/`수요일` 중 하나이고 값은 **정수 목록**(예: `[19]`)
- `validate_settings` 안에서 `fac == "올림픽공원레슨"`이면 위 함수로 검증하고 `continue`.
- `DEFAULT_SETTINGS`(폴백 기본값)에도 동일 블록 추가:
  `"올림픽공원레슨": {"받기": True, "코트": ["실외", "실내"], "주중": [19]}`.

> 설정 형식이 틀려도 봇은 멈추지 않는다(기존 폴백: `settings.last_good.yaml` → 코드 기본값).

### 5.4 `src/state.py` — 올림픽 직전 상태 저장/로드 (신규 함수)

칸별 상태 dict(`{"주중 실외 19시": "열림:19", …}`)를 작은 JSON으로 저장:
```python
save_olympic_state(path, state: dict)   # olympic_state.json
load_olympic_state(path) -> dict         # 없거나 깨졌으면 {}
```
(기존 `save_status`/`load_status`와 같은 dict-JSON 패턴. 이름만 용도에 맞게 둔다.)

### 5.5 `src/notifier.py` — 대기 알림 문구 (신규 함수)

```python
def format_olympic_alert(label: str, opened: bool, number: str = "") -> str:
    """label 예: '주중 실외 19시'. opened=True(열림)/False(다시 마감)."""
```
- 열림: `🎾 올림픽공원 테니스 레슨 대기 열림!\n📋 {label} (지금 '{number}')\n👉 지금 대기 신청: {OLYMPIC_URL}`
- 마감: `🔒 올림픽공원 레슨 대기 다시 마감\n📋 {label}`

`OLYMPIC_URL`은 `config.py`에서 가져온다(강남 `RESERVE_URL`과 별개 링크).

### 5.6 `src/main.py` — 감시 추가

`run_vacancy_alert()`와 별개로 **`run_olympic_alert()`** 를 새로 만들어 `main()`에서 호출:
```python
def run_olympic_alert():
    settings, _ = load_settings()
    current = fetch_olympic_states(settings)   # dict / None(실패) / {}(끔)
    if current is None:                        # 조회 실패 → 직전 유지 + 실패 누적
        failures = load_failures(FAIL_PATH)
        failures["올림픽공원레슨"] = failures.get("올림픽공원레슨", 0) + 1
        save_failures(FAIL_PATH, failures)
        return
    previous = load_olympic_state(OLYMPIC_STATE_PATH)
    is_first = not Path(OLYMPIC_STATE_PATH).exists()
    if not is_first:
        for text in build_olympic_messages(current, previous):  # 전이별 문구(7절)
            send_telegram(text)
    save_olympic_state(OLYMPIC_STATE_PATH, current)
```
- 상수: `OLYMPIC_STATE_PATH = "olympic_state.json"`.
- `main()`에 `run_olympic_alert()` 한 줄 추가(빈자리·신청 감시 뒤).
- **첫 실행은 저장만**(알림 없음) — 기존 부품과 동일한 폭탄 방지.

## 6. 데이터 흐름

```
(매 5분 실행) main.run_olympic_alert:
   fetch_olympic_states(settings)
     ├─ 받기 off/대상 0 → {} (조용, 저장만)
     ├─ 조회 실패 → None → 실패 누적 + 직전 상태 그대로(가짜 알림 방지)
     └─ 성공 → 표 GET → parse_olympic(targets) → {칸: 열림:n / 닫힘}
   직전 상태(load)와 비교 → build_olympic_messages(전이 감지)
     ├─ 닫힘→열림 : "🎾 대기 열림!" 1통
     ├─ 열림→닫힘 : "🔒 다시 마감" 1통
     └─ 그대로/숫자만 변함 : 조용
   현재 상태 저장(save) → 캐시에 보관 → 다음 실행이 이어서 비교
```

## 7. 상태 전이 판정 (마감↔숫자)

**`build_olympic_messages(current, previous) -> list[str]`** (순수함수, `differ.py` 또는 `olympic.py`):
- 각 칸 label에 대해 **열림/닫힘만** 비교(숫자 값은 무시) → `_is_open(v) = v.startswith("열림")`.
  - `previous 닫힘 → current 열림` : 열림 문구(현재 숫자 포함)
  - `previous 열림 → current 닫힘` : 마감 문구
  - 둘 다 열림(숫자만 3→2 변동) / 둘 다 닫힘(`마감↔X`) : **알림 없음**(사용자 확정: 조용)
- **직전 기록에 없던 칸(previous에 label 없음)** = 첫 등장 → 전이로 보지 않음(첫 실행 폭탄 방지,
  `find_opened`와 같은 방어). 단 `run_olympic_alert`가 `is_first`면 통째로 건너뛰므로 이 방어는 이중 안전.

> 숫자를 무시하고 열림/닫힘만 비교하므로, 대기 인원이 `19→17`처럼 흔들려도 재알림하지 않는다
> (사용자 확정: "열려 있는 동안 계속 알림 아님"). 알림 문구엔 그 순간 숫자를 참고로 보여준다.

## 8. 에러 처리

- 페이지 조회/파싱 실패 → `fetch_olympic_states`가 `None` → `failures["올림픽공원레슨"]` 누적 +
  **직전 상태 유지**(저장 안 함), 다른 부품(강남 등)은 정상 진행.
- 실패는 매일 아침 요약에 한 줄 보고(기존 `format_summary(failures=…)` 재사용 — 키만 추가되면 자동 표시).
- 표 모양이 예상과 다르면(칸 못 찾음) 해당 칸은 결과에서 빠질 뿐, 예외로 봇 전체를 멈추지 않음.

## 9. 테스트 (TDD)

- `build_targets`: 받기off→[], `코트:[실외,실내]`+`주중:[19]`→2개, 주말 추가 시 늘어남, 잘못된 키 무시.
- `parse_olympic`(저장한 실제 HTML 조각 사용):
  - 주중 실외/실내 19시 = 마감 → `"닫힘"` 2개.
  - 숫자로 바꾼 조각 → `"열림:<숫자>"`.
  - `X` 칸 → `"닫힘"`. 대상 아닌 칸은 결과에 없음. 시간→열번호 지도가 열 위치 바뀜에 견딤.
- `build_olympic_messages`(순수함수, 전이표):
  - 닫힘→열림=열림문구 / 열림→닫힘=마감문구 / 열림→열림(숫자만 변동)=없음 / 닫힘→닫힘=없음 /
    previous에 없던 칸=없음(첫 등장).
- `fetch_olympic_states`: 받기off→{}, 네트워크 예외→None(안 던짐), `requests` monkeypatch.
- `save/load_olympic_state`: tmp_path 왕복, 없는 파일→{}.

## 10. 범위·안전

- **읽기 전용**. 기존 부품(강남·esongpa·daechi·필터·differ) **무수정**
  (state·notifier·main·config·settings_loader에는 **함수/블록 추가만**, 기존 로직 변경 없음).
- 국민체육진흥공단은 대형 공공 사이트라 5분 주기(하루 ~288회) 조회에 무리 없음
  (강남과 동급). 별도 조회 게이트(대치유수지식 저빈도)는 지금 불필요 → YAGNI.
- 세익 재고 앱과 **완전히 무관**한 개인용 봇(별도 폴더·독립 git).
- **배포는 별도**: GitHub push 전 사용자 승인(push해야 실제 가동). 캐시 파일 추가 포함.
