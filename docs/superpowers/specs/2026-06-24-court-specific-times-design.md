# 코트별 추가 시간대 설계

> 날짜: 2026-06-24
> 상태: 설계 승인 완료 → 구현 진행

## 배경

현재 `settings.yaml`은 **시설(강남/송파/잠실) 단위**로만 시간대를 정한다(`강남: 매일 [19,20,21]`).
그래서 같은 강남 안의 포이 코트A와 코트B를 다른 시간대로 받을 수 없다.
(이전 "코트A 토요일 7시" 요청이 막혔던 구조적 한계.)

## 설계: 공통 + 코트별 "추가"

시설 공통 시간대는 그대로 두고, **특정 코트에만 시간을 더한다**(덮어쓰기가 아니라 추가).
예외 코트는 공통 시간대 **또는** 그 코트의 추가 시간대에 빈자리가 들어가면 알림.

### 설정 구조 (`settings.yaml`)

```yaml
강남:
  받기: true
  매일: [19, 20, 21]        # 모든 강남 코트 공통 (지금과 동일)
  코트추가:                  # 특정 코트에만 더하는 시간 (없으면 공통만)
    포이 코트A:
      평일: [7]             # 코트A = 공통 저녁 + 평일 아침 7시
    세곡 2번코트:
      주말: [12, 13]        # 세곡 2번 = 공통 저녁 + 주말 점심
```

- 코트 식별 키: `"{slot.court} {slot.place}"` 형식 (예: `"포이 코트A"`, `"세곡 2번코트"`).
- 각 코트 설정은 시간 키만 가진다(`받기` 없음 — 공통의 `받기`를 따른다).

### 요일 키 (묶음 키 추가)

기존 `매일`·`월`~`일`에 더해 **`평일`(월~금)·`주말`(토·일)** 묶음 키를 새로 추가한다.
"평일 아침 7시"를 한 줄로 적기 위함.

시간 해석 우선순위(`_hours_for`): **요일(월~일) > 평일/주말 > 매일**.
- 예: `{평일: [7], 매일: [19]}` → 월요일 `[7]`, 일요일 `[19]`(평일 아님·주말 없음→매일).

### 판정 규칙 (`is_wanted_for`)

```
받기=False면 제외.
빈자리 시각이
  ① 시설 공통 시간대(_hours_for(fac_cfg, 요일))에 있거나
  ② 그 코트의 추가 시간대(_hours_for(코트추가[코트키], 요일))에 있으면
→ 알림 대상.
코트추가에 없는 코트는 ①만(지금과 동일).
```

## 바뀌는 파일

1. `src/settings_loader.py` — `WEEKDAY_KEYS` 유지 + `TIME_KEYS`(요일+매일+평일/주말) 신설. 검증에 `평일`/`주말`/`코트추가` 허용 추가. `코트추가`는 dict(코트키→시간설정).
2. `src/filters.py` — `_hours_for(cfg, day_key)` 헬퍼(요일>평일/주말>매일) 추출 + `is_wanted_for`에 코트추가(②) 반영. 공통(①)도 `_hours_for`로 통일.
3. `settings.yaml` — `코트추가` 사용법 주석 + (선택) 실제 코트A 평일 7시 등록(배포 시 사용자 확인).
4. 테스트 — `_hours_for`(요일/평일/주말/매일 우선순위), `is_wanted_for`(공통/추가/둘 다/없는 코트), settings_loader 검증.

## 적용 범위 / 안전성

- **강남(포이·세곡)** 중심. 송파·잠실은 코트가 단일(`테니스장`)이라 코트추가가 거의 무의미하지만, 같은 `is_wanted_for`를 쓰므로 구조상 동일하게 동작(설정 안 하면 영향 0).
- 봇의 다른 부분(조회·재시도·알림·연속실패)은 **무수정**.
- `코트추가` 없는 기존 `settings.yaml`은 **동작 100% 동일**(하위호환). 형식 오류 시 settings_loader 폴백.
- 되돌리기: `git revert` 한 번.
