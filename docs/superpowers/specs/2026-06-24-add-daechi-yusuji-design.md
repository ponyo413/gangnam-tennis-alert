# 대치유수지 테니스장 추가 — 설계서

> 작성일: 2026-06-24
> 한 줄 요약: 강남·송파·잠실에 이어 **대치유수지 테니스장**의 빈자리(취소표)도
> 텔레그램으로 알려준다. 기존 부품은 손대지 않고 전용 부품 하나만 새로 얹는다.

---

## 1. 목적

대치유수지 테니스장(`대치유수지.kr`)의 빈자리가 생기면 텔레그램으로 알림.
강남(포이·세곡)·송파·잠실과 **같은 봇·같은 알림창**에서 함께 본다.

## 2. 확정 요구사항 (사용자 협의 완료)

| 항목 | 내용 |
|------|------|
| 감시 시간대 | **평일(월~금) 19시** / **토요일 7·9·19시** / **일요일 없음** |
| 코트 | A·B·C **전부** (어느 코트든 비면 알림) |
| 동작 | 읽기 전용 — **자동 예약 안 함** (강남과 동일) |
| 알림 | 기존 텔레그램 봇에 합류(별도 알림창 X) |

> 시간은 2시간 단위 칸의 **시작 시각**이다. 19시 = `19:00~21:00`, 7시 = `07:00~09:00`.

## 3. 사이트 분석 결과 (실측)

- 조회 주소: `https://www.대치유수지.kr/?act=reservation.reservation_list&type=8&cyear=YYYY&cmonth=M`
  - `type=8` = 테니스장 (7=축구장 등 다른 종목)
  - 퓨니코드 도메인: `xn--vk1b79znxd34c61h.kr`
- **로그인 불필요** — 누구나 빈자리표를 볼 수 있음 (송파·잠실보다 쉬움, 강남과 같은 급)
- **서버 렌더링 HTML** — 빈자리 데이터가 페이지에 바로 들어있음 (JavaScript/AJAX 아님)
- **봇 차단 없음** — 정상 응답(HTTP 200)
- 시간대: **2시간 단위 7칸** — `07~09, 09~11, 11~13, 13~15, 15~17, 17~19, 19~21`
- 코트: **A·B·C 3개**
- 빈자리 표시:
  - `possible_icn_on.gif`(title="가능") = **빈자리**
  - `possible_icn_off.gif`(title="불가능") = 찼거나 막힘
- 빈자리 칸 구조(결정적):
  ```html
  <dt><a href="#" class="_rev" data-date="2026-07-01" data-time="0" data-type="9">
      <img src="/images/sub/possible_icn_on.gif" title="가능" /></a></dt>
  ```
  - `data-date` = 날짜(`YYYY-MM-DD`), `data-time` = 시간 인덱스(0~6)
  - **시작시각 = 7 + (data-time × 2)** → 0→07시, 1→09시 … 6→19시
  - 코트는 한 행(`<dl>`)의 `<dt>` **순서**(1번째=A, 2번째=B, 3번째=C)로 판별
  - 빈자리(`on`) 칸에만 `data-date`/`data-time`이 있고, 찬 칸엔 없음 → 빈자리만 깔끔히 추출

## 4. 접근법 — 전용 부품 신설 (`src/daechi.py`)

봇은 "**시설군마다 자기 부품 하나**" 구조다.
- 강남(포이·세곡): `src/fetcher.py` (강남구청 REST API)
- 송파·잠실: `src/esongpa.py` (로그인 + HTML 파싱)

대치유수지는 사이트 방식이 위 둘과 모두 달라서(로그인 없는 Rhymix 계열 HTML),
**`src/daechi.py`** 를 새로 만들어 나란히 둔다.

> 대안 비교 — 강남 부품에 합치기: 사이트 구조가 완전히 달라 한 파일이 지저분해짐(기각).
> esongpa 부품에 넣기: 거긴 로그인 전제라 안 맞음(기각). → **신규 부품이 정답.**

파싱·필터 패턴은 **esongpa 방식**을 따른다(설정표 cfg를 받아 내부에서 시간 필터까지 수행).

## 5. 컴포넌트 설계

### 5.1 `src/daechi.py` (신규)

```
fetch_daechi_slots(settings) -> list[Slot]
```
- `settings["대치유수지"]` 설정을 읽음. `받기=False`거나 설정 없으면 빈 목록(비활성).
- (esongpa와 달리 닫힘 처리가 없으므로 `previous`(직전 빈자리) 인자를 받지 않는다 — 군더더기 제거.)
- **이번 달 + 다음 달** 두 페이지를 GET (`cyear`/`cmonth` 파라미터, 로그인 없음).
- `parse_daechi(html)` 로 빈자리 칸을 `Slot` 목록으로:
  - `Slot(court="대치유수지", place="A코트"/"B코트"/"C코트", date, time)`
  - (Slot의 `court`=시설명, `place`=세부 코트 — 기존 모델 그대로)
- 각 슬롯에 `is_wanted_for(slot, cfg)`(공용 필터) + **미래 시각만** 적용.
- 한 달/시설 조회 실패는 건너뛰고 나머지는 계속(기존 패턴).

보조 함수(작게 분리):
- `parse_daechi(html)` — `<dl>` 행 단위로 `<dt>` 3개를 A/B/C에 매핑, `on`+`data-date` 칸만 추출
- 시간 인덱스 → 시작시각 변환(`7 + idx*2`)

### 5.2 `settings.yaml` — 대치유수지 블록 추가

```yaml
대치유수지:
  받기: true        # 끄려면 false 로만 바꾸면 끝
  평일: [19]        # 월~금: 저녁 7시(19~21시)
  토: [7, 9, 19]    # 토요일: 오전 7·9시 + 저녁 7시
  # 일요일은 줄 없음 = 감시 안 함
```
> `settings.last_good.yaml`(백업본)은 다음 정상 실행 때 자동 갱신되므로 수동 수정은 불필요하다.

### 5.3 `src/main.py` — 호출 추가 (2곳)

`run_vacancy_alert()`와 `run_summary()`에서 esongpa 호출 옆에 한 줄씩 추가:
```python
wanted += fetch_daechi_slots(settings)
```
실패해도 다른 시설 알림은 계속 + `failures`에 누적(기존과 동일한 try/except 패턴).

### 5.4 수정 불필요 확인(조사 완료)

- `settings_loader.py`: 시설명 화이트리스트가 **없음**(시간 키만 검증) → 대치유수지 추가만으로 통과. **수정 불필요.**
- `notifier.py`의 `format_summary`: `🏟 {court} {place} 📅 {date}(요일) {time}` 형식이라 대치유수지도 자동 표시. **수정 불필요.**

## 6. 데이터 흐름

```
대치유수지 페이지(이번달·다음달) GET
   → parse_daechi: 빈자리 칸 → Slot(대치유수지, A/B/C코트, 날짜, 시각)
   → is_wanted_for(설정 시간대) + 미래만 필터
   → (강남·송파·잠실 결과와 합침)
   → differ: 직전 빈자리와 달라졌으면
   → 텔레그램으로 현재 전체 현황 1통 발송
```

## 7. 접수시간(닫힘) 처리 — 불필요

잠실은 저녁에 모든 칸이 "접수불가"로 잠기는 특수 사이트라 직전 유지 로직이 있었다.
대치유수지는 실측상 미래 날짜에 빈자리(`on`)가 그대로 보이는 **강남식 단순 구조**이므로,
잠실식 닫힘 처리는 두지 않고 **미래 시각 필터만** 적용한다.
(구현 단계에서 저장한 실제 HTML로 한 번 더 확인한다.)

## 8. 에러 처리

- 한 시설(대치유수지) 조회 실패 → 그 시설만 건너뜀, 강남·esongpa 알림은 정상 진행.
- 실패는 `failures.json`에 누적 → 매일 아침 요약에 한 줄 보고(기존 구조 재사용).

## 9. 테스트 (TDD)

- 실측 저장본(대치유수지 7월 페이지 HTML)을 **테스트 fixture**로 사용.
- 검증 항목:
  1. `parse_daechi` — 빈자리 칸만 추출(찬 칸 제외)
  2. 코트 매핑 — `<dt>` 순서 → A/B/C 정확
  3. 시간 매핑 — `data-time` 0~6 → 07~19시(특히 데이터에 없는 19시도 합성으로 검증)
  4. 미래 필터 — 지난 시각 제외
  5. `is_wanted_for` 연동 — 평일 19 / 토 7·9·19 / 일요일 제외가 의도대로

## 10. 범위·안전

- **읽기 전용**, 기존 부품(강남·esongpa·필터·알림) **무수정**, 부품 추가만.
- 세익 재고 앱과 **완전히 무관**한 개인용 봇(별도 폴더·독립 git).

### 후속(이번 범위 밖, 선택)
- 알림 끝의 예약 링크(`RESERVE_URL`)는 현재 강남 1개 공용. 요약 1통에 여러 시설이 섞일 때
  시설별 링크가 필요하면 추후 개선(지금은 기존과 동일하게 둔다).
