# 강남 테니스장 취소표 알림 시스템 — 설계서

> 작성일: 2026-06-22
> 한 줄 요약: 강남구 예약 사이트를 24시간 자동으로 들여다보며, 포이·세곡 테니스장에
> 원하는 시간대 빈자리(취소표)가 생기면 **텔레그램으로 즉시 알려주는** 개인용 시스템.

> ⚠️ 이 프로젝트는 **세익(SEIK) 재고관리 앱과 완전히 별개**입니다.
> 새 폴더(`바탕화면\gangnam-tennis-alert`)에서 따로 만들며, 세익 git 저장소와 섞이지 않습니다.

## 1. 배경 — 왜 만드는가

강남구 공공 테니스장(포이·세곡)은 예약 경쟁이 치열해서, 한 번 마감되면
누군가 **취소할 때 생기는 빈자리**를 노려야 한다. 그런데 그 취소표는 보통
몇 분 안에 다른 사람이 채간다. 사람이 하루 종일 사이트를 새로고침할 수는 없으니,
**컴퓨터가 대신 24시간 감시하다가 빈자리가 뜨면 즉시 알려주는** 시스템이 필요하다.

### 중요한 사실 (조사로 확인됨)

- 공공데이터포털(data.go.kr)의 "강남구 테니스장 현황" 파일은 **실시간이 아니다.**
  가끔 갱신되는 "시간표 사진"이라 취소표를 잡는 데 부적합.
- 진짜 빈자리는 **강남구 통합 예약 사이트(life.gangnam.go.kr)**에서만 실시간으로 확인 가능.
- 그 사이트의 자동 접근 규칙(robots.txt)은 일반 접근을 **허용**한다 (`User-agent: * / Allow: /`).
- 빈자리 시간표는 "센터→시설→날짜를 고르고 조회 버튼을 눌러야" 나오는
  **자바스크립트 동적 방식**이다. → 빈자리를 읽어오는 방법을 구현 1단계에서 직접 확인해야 함.

## 2. 확정된 요구사항

| 항목 | 결정 내용 |
|------|-----------|
| 감시 대상 | 포이테니스장, 강남세곡체육공원 테니스장 |
| 원하는 시간대 | **평일 저녁**(월~금 18:00~22:00) + **주말**(토·일 06:00~22:00) ※ 기본값, 나중에 쉽게 조정 |
| 알림 도구 | **텔레그램** (무료·즉시 푸시) |
| 알림 시간 | **24시간** (한밤중에도 빈자리 뜨면 바로 알림) |
| 시스템 범위 | **알림만.** 자동 예약(매크로)은 하지 않음 — 약관 위반·타인 피해 소지 차단 |
| 실행 장소 | **무료 클라우드(GitHub Actions)** — 사용자 PC를 꺼도 24시간 작동 |
| 확인 주기 | **3~5분 간격** (사이트에 부담 주지 않는 선) |

## 3. 시스템 구조

```
┌──────────────────────────────────────────────────────────┐
│  무료 클라우드(GitHub Actions) — 24시간 깨어있는 "경비원"      │
│                                                           │
│   3~5분마다 실행:                                          │
│                                                           │
│   ① 조회기(Fetcher)                                        │
│      강남구 예약사이트에서 포이·세곡의                        │
│      "평일 저녁 + 주말" 빈자리 목록을 읽어옴                   │
│                  │                                        │
│                  ▼                                        │
│   ② 비교기(Differ)                                         │
│      클라우드에 저장해 둔 "직전 빈자리 목록"과 비교            │
│      → 이번에 새로 생긴 빈자리만 골라냄                       │
│                  │                                        │
│         ┌────────┴────────┐                               │
│      새 빈자리 없음     새 빈자리 있음!                       │
│         │                  │                              │
│      조용히 종료     ③ 알림기(Notifier)                     │
│                         텔레그램으로 메시지 발송 📲           │
│                  │                                        │
│                  ▼                                        │
│   ④ 현재 빈자리 목록을 "직전 목록"으로 저장 (다음 비교용)       │
└──────────────────────────────────────────────────────────┘
```

### 구성 요소 3개 (각각 한 가지 일만)

1. **조회기(Fetcher)** — 사이트에서 빈자리 목록을 읽어 정리된 데이터로 돌려준다.
   입력: 없음 / 출력: `[{코트, 날짜, 시간}, ...]`
2. **비교기(Differ)** — "이번 목록"과 "직전 목록"을 비교해 **새로 생긴 빈자리만** 돌려준다.
   입력: 이번 목록 + 직전 목록 / 출력: 새 빈자리 목록
3. **알림기(Notifier)** — 새 빈자리 목록을 받아 텔레그램 메시지로 만들어 보낸다.
   입력: 새 빈자리 목록 / 출력: 텔레그램 발송

> 셋을 분리하는 이유: 각각 따로 테스트할 수 있고, 한 곳이 고장 나도 원인을 빨리 찾는다.
> (예: 알림이 안 오면 → 조회기 문제인지 알림기 문제인지 따로 확인 가능)

---

## 4. 동작 흐름 (한 번 실행될 때)

1. 조회기가 사이트에서 포이·세곡 빈자리를 모두 읽는다.
2. "평일 저녁 + 주말" 조건에 맞는 것만 거른다.
3. 비교기가 직전 목록과 비교해 **새로 생긴 빈자리**를 찾는다.
4. 새 빈자리가 있으면 알림기가 텔레그램으로 보낸다. 없으면 아무것도 안 한다.
5. 이번 목록을 "직전 목록"으로 저장하고 종료한다.

---

## 5. 핵심 기술 결정 — 빈자리를 "어떻게" 읽나 (가장 불확실한 부분)

사이트가 자바스크립트로 빈자리를 보여주므로, 구현 1단계에서 다음 두 길 중 하나를 정한다:

- **A안 — 내부 통로(API) 직접 호출 (선호):**
  브라우저 개발자도구의 "네트워크" 탭으로, 조회 버튼을 누를 때 사이트가
  어떤 주소에 무슨 정보를 주고받는지 찾는다. 찾으면 그 주소로 직접 요청해
  빈자리 데이터(JSON)만 깔끔하고 빠르게 받는다.
- **B안 — 가짜 브라우저 자동화 (대안):**
  A안이 막히면, 자동 브라우저(Playwright)를 띄워 사람처럼
  센터·시설·날짜를 선택하고 조회 버튼을 눌러 화면에서 빈자리를 읽는다.
  조금 무겁지만 거의 확실하게 동작한다.

→ **1단계 산출물:** "A안 가능 / B안으로 진행" 결론 + 빈자리를 실제로 읽어내는 코드.

## 6. 데이터 — 무엇을, 어떻게 저장하나

- **빈자리 한 칸의 형태:** `{ 코트: "포이", 날짜: "2026-06-25", 시간: "19:00" }`
- **"직전 목록" 저장 위치:** 클라우드(GitHub Actions)는 실행할 때마다 기억이 초기화되므로,
  직전 빈자리 목록을 **저장소 파일 또는 Actions 캐시**에 보관해 다음 실행과 비교한다.
  (구현 시 가장 단순하고 안정적인 방법 선택 — 상세는 구현 계획서에서)
- 개인정보·비밀번호 저장 없음. 텔레그램 토큰만 **금고(GitHub Secrets)**에 보관.

## 8. 예의·안전·법적 고려

- ✅ robots.txt가 자동 접근을 허용함을 확인.
- ✅ **3~5분 간격**으로만 조회 — 사이트 부담·차단 방지.
- ✅ **개인용·알림만** — 자동예약 매크로 아님(타인 피해·약관 위반 소지 차단).
- ✅ 텔레그램 토큰은 코드에 적지 않고 **GitHub Secrets(금고)**에 보관.
- ✅ 사이트가 명시적으로 거부 신호(차단·이용약관 변경)를 보이면 즉시 중단·재검토.

## 9. 오류·예외 처리

| 상황 | 대응 |
|------|------|
| 사이트 일시적 먹통/응답 지연 | 이번 차례는 조용히 건너뛰고 다음 차례에 재시도 |
| 사이트 개편으로 빈자리를 못 읽음 | **"읽기 실패" 경고를 텔레그램으로 발송** (조용히 죽지 않게) |
| 텔레그램 발송 실패 | 다음 실행 때 재시도, 로그 남김 |
| 같은 빈자리 중복 알림 | 비교기가 "직전 목록"과 대조해 새 것만 보내므로 자연히 방지 |

## 10. 기술 스택

- **언어:** Python (이런 조회·알림 자동화에 코드가 짧고, GitHub Actions와 궁합 좋음)
- **클라우드:** GitHub Actions (무료, 신용카드 불필요, 코드와 실행이 한 곳에)
- **알림:** 텔레그램 Bot API
- **빈자리 읽기:** A안(`requests`로 내부 API 호출) 또는 B안(`Playwright` 브라우저 자동화)

---

## 11. 구현 단계 (작게 시작 → 확인 → 키우기)

1. **1단계 — 빈자리 읽기 성공시키기:** A안/B안 결정 + 포이·세곡 빈자리를 화면에 출력.
2. **2단계 — 텔레그램 알림 연결:** 빈자리 목록을 텔레그램으로 보내기.
3. **3단계 — "새 빈자리만" 골라내기:** 직전 목록과 비교(비교기) + 시간대 필터.
4. **4단계 — 클라우드 24시간 자동화:** GitHub Actions에 올려 3~5분마다 자동 실행.

각 단계마다 동작을 직접 확인한 뒤 다음으로 넘어간다.
